Genuine card numbers will produce an error in your test environment. To simulate payments, use any of the following test card details provided in the tables below.

<TipBox variant="info" title="Expiry date">

Use any _future_ expiry date (`mm/yy`) with our test cards.

</TipBox>

## Non-3D Secure enabled cards

From the Hub, if the _API Status & Keys_ of the channel is:
* Live: With `"3ds.enabled": true`, the cards in the table below will return `20150 - Card not 3D Secure (3DS) enabled` soft declined code. With `"attempt_n3d": true` in the request body, the 3DS payment will be fallback to non-3DS.
* Test: With `"3ds.enabled": true`, using the cards from our [3DS testing page](/testing/3d-secure-testing), the `Card not enrolled` or the `Issuer not enrolled` cards will return `20150 - Card not 3D Secure (3DS) enabled` soft declined code. 
With `"attempt_n3d": true`, the 3DS payment will be fallback to non-3DS.
